# Method Compatibility Matrix

This document is the method-level contract for backend compatibility, routing
mode, fallback behavior, and semantic risk. It is generated from
[capabilities.yaml](capabilities.yaml).

It focuses on routed, downgraded, bridged, or risk-bearing RPC families rather
than every overlapping Apache HMS method. Front-door-local security token APIs
remain documented in [README.md](README.md).

Status legend:

- `supported`: normal path for that backend/runtime combination
- `degraded`: works through a downgrade, default-catalog pin, or compatibility response
- `rejected`: fails explicitly instead of guessing
- `unsafe`: can route, but multi-catalog semantics are not production-safe until validated

Refresh the generated table with:

```bash
mvn -o -q -Dtest=CapabilityMatrixDocSyncTest -Dcapabilities.updateReadme=true test
```
